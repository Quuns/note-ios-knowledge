## 摘要
本节课原计划继续讲 iOS 内存管理中的 SideTable，但由于直播网络不稳定，课程中途调整为离屏渲染专题讲解。内容围绕 iOS 图形渲染流程、CPU 与 GPU 的分工、帧缓冲区、VSYNC、掉帧原因，以及哪些 UIKit/Core Animation 操作会触发离屏渲染展开，并结合代码演示了光栅化、遮罩、阴影、抗锯齿、透明度、圆角等场景的触发条件与优化方式。


### 亮点
- 🎯 iOS 内存管理中常见的三种方案包括 Tagged Pointer、nonpointer isa 和 SideTable，它们分别服务于小对象优化、指针信息压缩和额外引用计数或弱引用管理。
- 🧠 SideTable 内部包含锁、引用计数表和弱引用表，其中锁用于保障多线程访问时的数据安全。
- 🔐 Apple 并不是只使用一张全局 SideTable，而是通过 StripMap 准备多张表并均匀分配对象，从而在内存占用和锁竞争性能之间取得平衡。
- 🖼️ 离屏渲染的前提是理解 iOS 图形渲染链路：UIKit 负责上层界面封装，Core Animation 负责动画与图层提交，底层再依赖 OpenGL ES 或 Metal 进行 GPU 渲染。
- ⚙️ CPU 主要负责视图创建、布局计算、图片编解码和纹理准备，而 GPU 主要负责顶点处理、像素颜色计算、光栅化和最终图像合成。
- 📺 渲染结果通常会写入帧缓冲区，显示控制器再按照固定刷新频率读取帧缓冲区数据并显示到屏幕上。
- ⏱️ VSYNC 是垂直同步信号，如果 CPU 和 GPU 没能在下一次 VSYNC 到来前完成一帧内容提交，就会发生掉帧并引起界面卡顿。
- 🧩 离屏渲染指的是渲染结果无法直接写入帧缓冲区，而是先写入额外的临时缓冲区，完成中间合成后再写回帧缓冲区。
- 🏔️ 当图层结构复杂时，GPU 可能无法一次性完成最终画面渲染，因此需要借助额外缓冲区分步合成，这就是离屏渲染产生的重要原因。
- 🟡 Xcode 的 Debug 选项中可以开启 Color Offscreen-Rendered，界面变黄通常表示该区域发生了离屏渲染。
- 🧱 shouldRasterize 会触发离屏渲染，但它本质上是一种缓存机制，会把图层以位图形式缓存起来以减少后续 GPU 计算。
- ⚠️ 光栅化缓存时间较短，且适用场景有限，因此不建议在面试或项目优化中泛泛地说“开启光栅化可以优化 UITableView 性能”。
- 🎭 mask 遮罩一定会增加图层合成复杂度，因为遮罩作为额外图层参与计算，通常会触发离屏渲染。
- 🌫️ shadow 阴影默认也容易触发离屏渲染，因为系统需要根据图层内容动态计算阴影范围。
- ✅ 阴影可以通过设置 shadowPath 优化，因为提前指定阴影路径后，系统不再依赖图层内容动态推导阴影形状，从而避免不必要的离屏渲染。
- 🔍 抗锯齿可能触发离屏渲染，尤其是在图片旋转、缩放比例不匹配时，GPU 需要额外计算边缘像素以消除锯齿。
- 📐 为了减少抗锯齿带来的额外开销，图片资源尺寸或比例应尽量与控件尺寸保持一致。
- 👥 allowsGroupOpacity 在父视图透明度不为 1 且存在子视图时，可能需要进行颜色混合计算，因此可能触发离屏渲染。
- 🔴 圆角是否触发离屏渲染不能简单判断，iOS 9 之后单独设置 cornerRadius 通常不会触发，但配合 masksToBounds 裁剪 contents 时就可能触发。
- 🏷️ UILabel 有特殊性，给 UILabel 设置 backgroundColor 实际更偏向影响 contents，因此同样的圆角代码在 UILabel 和 UIImageView 上可能表现不同。
- ✂️ 如果需要给图片做圆角，可以考虑使用贝塞尔曲线预处理图片，或者让设计切一张遮罩图覆盖在头像上，这通常比运行时裁剪更高效。
- 🧾 重写 drawRect: 会创建额外的 backing store，并通过 Core Graphics 在 CPU 上绘制，因此它也可以被理解为一种特殊的离屏渲染形式。
- 🚫 日常开发中应尽量避免不必要的 drawRect: 绘制、动态遮罩、未优化阴影和复杂透明图层混合，以降低内存消耗和掉帧风险。

### 思考 
1. **[什么是离屏渲染，为什么它会导致卡顿？](https://bibigpt.co/search?q=%E4%BB%80%E4%B9%88%E6%98%AF%E7%A6%BB%E5%B1%8F%E6%B8%B2%E6%9F%93%EF%BC%8C%E4%B8%BA%E4%BB%80%E4%B9%88%E5%AE%83%E4%BC%9A%E5%AF%BC%E8%87%B4%E5%8D%A1%E9%A1%BF%EF%BC%9F)**
  - 离屏渲染是指图层内容无法直接渲染到帧缓冲区，而需要先渲染到额外的临时缓冲区，完成合成后再写回帧缓冲区。这个过程会增加 GPU/CPU 工作量和内存消耗，如果一帧内容无法在下一次 VSYNC 到来前完成提交，就会掉帧，从而造成卡顿。
- 2. **[设置圆角一定会触发离屏渲染吗？](https://bibigpt.co/search?q=%E8%AE%BE%E7%BD%AE%E5%9C%86%E8%A7%92%E4%B8%80%E5%AE%9A%E4%BC%9A%E8%A7%A6%E5%8F%91%E7%A6%BB%E5%B1%8F%E6%B8%B2%E6%9F%93%E5%90%97%EF%BC%9F)**
  - 不一定。iOS 9 之后，单独设置 cornerRadius 通常不会触发离屏渲染；但如果同时涉及 contents 裁剪，例如设置 masksToBounds 或 clipsToBounds，并且图层内容需要被裁剪，就可能触发离屏渲染。UILabel 还存在特殊性，同样代码在 UILabel 和 UIImageView 上表现可能不同。
- 3. **[阴影为什么会触发离屏渲染，应该如何优化？](https://bibigpt.co/search?q=%E9%98%B4%E5%BD%B1%E4%B8%BA%E4%BB%80%E4%B9%88%E4%BC%9A%E8%A7%A6%E5%8F%91%E7%A6%BB%E5%B1%8F%E6%B8%B2%E6%9F%93%EF%BC%8C%E5%BA%94%E8%AF%A5%E5%A6%82%E4%BD%95%E4%BC%98%E5%8C%96%EF%BC%9F)**
  - 阴影默认需要系统根据图层内容动态计算阴影范围，这会增加图层合成复杂度，因此容易触发离屏渲染。优化方式是提前设置 shadowPath，明确告诉系统阴影路径，使阴影可以独立渲染，减少动态计算和离屏渲染开销。


### 术语解释
- **离屏渲染（Offscreen Rendering）**: 渲染结果不能直接写入帧缓冲区，而是先写入额外缓冲区进行中间合成，之后再写回帧缓冲区的过程。
- **帧缓冲区（Frame Buffer）**: 用于存储一帧屏幕像素数据的内存区域，显示控制器会从这里读取数据并显示到屏幕上。
- **VSYNC（垂直同步信号）**: 显示器完成一帧扫描并准备开始下一帧时产生的同步信号，通常用于衡量屏幕刷新节奏和掉帧情况。
- **光栅化（Rasterization）**: 将图形转换成屏幕像素的过程；在 Core Animation 中，shouldRasterize 还表示将图层缓存为位图以复用渲染结果。
- **Backing Store**: 图层背后的位图缓存区域，用于存放绘制结果；重写 drawRect: 时可能额外创建 backing store，从而增加内存消耗。
