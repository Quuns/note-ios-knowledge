## 摘要
本节课主要围绕 Objective-C 类的底层内存结构展开，从断点调试问题切入，讲解了关闭 LTO 优化以便调试源码；随后通过 LLDB、源码结构体和内存读取，验证了实例对象、类对象、元类、根元类之间的 `isa` 指向关系，以及类与元类的继承链。课程后半部分重点分析 `objc_class` 的结构布局，说明如何通过内存平移读取 `bits` 数据，并进一步探索 `class_rw_t` 中存储的实例方法列表与属性列表，为下一节课继续分析成员变量、类方法、`class_ro_t/class_rw_t` 等内容做铺垫。


### 亮点
- 🛠️ 课程首先解决了源码中无法命中断点的问题，原因是工程开启了 Link Time Optimization，需要在 Build Settings 中搜索 LTO 并关闭该优化。
- 🧩 Objective-C 的实例对象内部主要包含 `isa` 指针和成员变量值，其中 `isa` 指针用于指向该对象所属的类对象。
- 🧠 通过类方法 `class`、实例对象的 `class` 方法以及 Runtime API `object_getClass` 获取到的类对象地址一致，从而证明类对象在内存中有且只有一个。
- 🏗️ 类对象本质上是 `objc_class` 结构体，而 `objc_class` 又继承自 `objc_object`，因此类本身也可以被视为一种对象。
- 🔁 实例对象的 `isa` 指向类对象，类对象的 `isa` 指向元类对象，元类对象的 `isa` 最终指向根元类。
- 🧭 根元类的 `isa` 指针指向它自己，这形成了 Objective-C 对象系统中非常关键的闭环结构。
- 🌳 类对象存在继承链，元类对象同样存在继承链，并且可以概括为“元类的父类就是父类的元类”。
- 🧱 `NSObject` 是根类，根元类的父类指向 `NSObject` 类对象，这也是 Objective-C 类体系设计中的特殊关系。
- 🔍 `objc_class` 结构体中包含 `isa`、`superclass`、`cache` 和 `bits` 等重要字段，其中 `bits` 关联了类的更多运行时数据。
- 📦 通过类对象首地址进行内存平移，可以访问到 `bits` 字段所在的内存区域，这依赖于结构体字段在内存中的连续布局。
- 🧮 课程通过数组和指针示例解释了内存平移，说明指针加一并不是固定加一个字节，而是根据指针指向的数据类型大小移动。
- 🧾 在读取 `bits` 后，需要结合源码中的数据结构和方法，例如 `data()`，进一步获取 `class_rw_t` 中保存的方法、属性和协议等信息。
- 🧪 通过 LLDB 一层层读取 `method_list_t`，可以看到类中保存的实例方法、属性自动生成的 getter/setter，以及用于释放成员变量的 `.cxx_destruct` 方法。
- 🏷️ 类方法不会出现在类对象的实例方法列表中，因为类方法存储在元类对象中，这一点需要通过后续探索继续验证。
- 📚 属性列表可以通过 `properties()` 相关路径读取出来，示例中成功读取到了 `name` 和 `age` 两个属性。
- 🔡 课程还补充讲解了大端和小端的概念，说明不同架构下读取 Runtime 数据时可能需要采用不同的数据读取方式。
- 🧑‍💻 老师强调，本节课的重点不只是结论，而是通过源码、LLDB 和内存分析一步步验证底层知识的探索过程。

### 思考 
1. **[为什么关闭 LTO 后才能正常打断点？](https://bibigpt.co/search?q=%E4%B8%BA%E4%BB%80%E4%B9%88%E5%85%B3%E9%97%AD%20LTO%20%E5%90%8E%E6%89%8D%E8%83%BD%E6%AD%A3%E5%B8%B8%E6%89%93%E6%96%AD%E7%82%B9%EF%BC%9F)**
  - 因为 LTO 是链接阶段优化，编译器可能会在链接时调整、合并或优化代码结构，导致源码位置与实际执行代码不完全对应，从而影响断点命中。关闭 Link Time Optimization 后，调试信息更稳定，断点更容易准确命中。
- 2. **[类对象和元类对象有什么区别？](https://bibigpt.co/search?q=%E7%B1%BB%E5%AF%B9%E8%B1%A1%E5%92%8C%E5%85%83%E7%B1%BB%E5%AF%B9%E8%B1%A1%E6%9C%89%E4%BB%80%E4%B9%88%E5%8C%BA%E5%88%AB%EF%BC%9F)**
  - 类对象主要保存实例相关的信息，例如实例方法、属性、协议等；元类对象则主要保存类本身相关的信息，例如类方法。实例对象的 `isa` 指向类对象，而类对象的 `isa` 指向元类对象。
- 3. **[为什么要通过 LLDB 和源码一步步验证这些结构？](https://bibigpt.co/search?q=%E4%B8%BA%E4%BB%80%E4%B9%88%E8%A6%81%E9%80%9A%E8%BF%87%20LLDB%20%E5%92%8C%E6%BA%90%E7%A0%81%E4%B8%80%E6%AD%A5%E6%AD%A5%E9%AA%8C%E8%AF%81%E8%BF%99%E4%BA%9B%E7%BB%93%E6%9E%84%EF%BC%9F)**
  - 因为 Runtime 的很多机制并不是只靠记忆就能真正理解的。通过 LLDB 查看内存地址、结合源码结构体分析字段布局，可以更直观地确认 `isa` 指向、继承链和方法列表存储位置，从而建立可靠的底层认知。


### 术语解释
- **isa 指针**: Objective-C 对象中的核心指针，用于指向对象所属的类对象；类对象中的 `isa` 则指向元类对象，根元类的 `isa` 指向自身。
- **类对象（Class Object）**: 描述某个类的运行时对象，本质上是 `objc_class` 结构体，通常保存实例方法、属性、协议等与实例相关的信息。
- **元类（Meta-class）**: 描述类对象自身的类，主要用于存储类方法；类对象的 `isa` 指针指向它对应的元类对象。
- **objc_class**: Objective-C Runtime 中表示类的数据结构，内部包含 `isa`、`superclass`、`cache`、`bits` 等字段，是理解类内存结构的关键入口。
- **内存平移**: 通过指针地址加偏移量访问后续内存区域的方式；偏移量通常与数据类型大小和结构体字段布局有关，本课用它来访问 `objc_class` 中的 `bits` 数据。
