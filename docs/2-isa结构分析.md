## 摘要
本节课围绕 Objective-C 对象底层结构继续展开，重点分析对象内存大小的影响因素、成员变量在对象内存中的分布、结构体与联合体的区别、位域的节省空间机制，以及 `isa` 指针如何通过 non-pointer isa 优化存储对象相关信息。课程还演示了如何利用 LLDB 指令和位运算从 `isa` 中还原类对象地址，并补充说明了 `new` 方法本质上等价于 `alloc + init`。

- 🧠 对象的内存大小主要由 `isa` 指针和成员变量决定，实例方法和类方法不会直接增加实例对象的内存占用。
- 📦 Objective-C 对象内部通常按照 8 字节对齐，而系统实际分配对象内存时通常按照 16 字节对齐。
- 🔍 属性本身不是直接影响对象内存的因素，真正影响对象大小的是属性自动生成的成员变量。
- 🧩 Apple 会对同一个类内部的成员变量顺序进行自动重排，从而减少内存对齐带来的空间浪费。
- 🧱 子类继承父类时，父类的数据结构是一块连续内存，子类不会随意打乱父类成员变量的布局。
- ⚠️ 虽然父类成员变量顺序可能影响子类对象大小，但日常开发中通常不需要为了几个字节刻意调整顺序。
- 🧮 位域可以指定某个字段只占用若干 bit 位，适合系统级源码中做精细化内存优化。
- 🔗 联合体中的多个成员共用同一块内存，因此同一时间通常只能正确使用其中一个成员值。
- 🏗️ 结构体的成员变量可以共存，而联合体成员变量互斥；联合体更节省空间，但使用场景更受限制。
- 🧭 `isa` 指针本质上在源码中以联合体形式存在，用于兼容普通指针和 non-pointer isa 两种存储方式。
- 🚀 non-pointer isa 是 Apple 的内存优化方案，它不会只存类对象地址，还会把引用计数、弱引用标记、析构标记等信息压缩进 `isa` 的 64 位空间。
- 🛠️ 不同 CPU 架构下，non-pointer isa 中各字段所占 bit 位不同，因此分析源码时必须区分真机、模拟器、ARM64、x86_64 等环境。
- 🧪 通过 LLDB 的 `p`、`po`、`x/4gx` 等指令，可以观察对象内存地址、成员变量分布和 `isa` 指针内容。
- 🧷 可以通过右移、左移或与掩码 `ISA_MASK` 的方式，从 `isa` 指针中提取类对象地址。
- 🔢 `extra_rc` 字段用于存储引用计数的一部分，不同架构下可存储的最大范围不同，超出后会借助其他结构存储。

### 思考 
1. **[Objective-C 对象的内存大小到底由什么决定？](https://bibigpt.co/search?q=Objective-C%20%E5%AF%B9%E8%B1%A1%E7%9A%84%E5%86%85%E5%AD%98%E5%A4%A7%E5%B0%8F%E5%88%B0%E5%BA%95%E7%94%B1%E4%BB%80%E4%B9%88%E5%86%B3%E5%AE%9A%EF%BC%9F)**
  - 主要由 `isa` 指针和成员变量决定。`isa` 指针通常占 8 字节，成员变量根据类型大小、对齐规则和编译器重排结果共同决定对象实际占用空间。实例方法和类方法不存放在实例对象内部，因此不会直接影响实例对象大小。
- 2. **[为什么 non-pointer isa 能优化内存？](https://bibigpt.co/search?q=%E4%B8%BA%E4%BB%80%E4%B9%88%20non-pointer%20isa%20%E8%83%BD%E4%BC%98%E5%8C%96%E5%86%85%E5%AD%98%EF%BC%9F)**
  - 因为 64 位环境下，`isa` 指针不需要完整 64 位都用来存储类对象地址。Apple 将部分 bit 位用于存储引用计数、是否有关联对象、是否有析构函数、是否被弱引用、是否正在释放等对象相关信息，从而避免额外开辟空间保存这些状态。
- 3. **[日常开发中需要手动调整成员变量顺序来优化内存吗？](https://bibigpt.co/search?q=%E6%97%A5%E5%B8%B8%E5%BC%80%E5%8F%91%E4%B8%AD%E9%9C%80%E8%A6%81%E6%89%8B%E5%8A%A8%E8%B0%83%E6%95%B4%E6%88%90%E5%91%98%E5%8F%98%E9%87%8F%E9%A1%BA%E5%BA%8F%E6%9D%A5%E4%BC%98%E5%8C%96%E5%86%85%E5%AD%98%E5%90%97%EF%BC%9F)**
  - 通常不需要。Apple 已经会对同一个类内部的成员变量进行自动重排，而且几个字节的节省在多数业务场景中意义很小。真正的内存优化更多发生在大对象、大图片、瞬时内存峰值、缓存策略和生命周期管理等方面。


### 术语解释
- **isa 指针**: Objective-C 对象中的核心指针，用于让实例对象找到它所属的类对象。现代运行时中，`isa` 往往不是单纯的类对象地址，而是经过优化的结构。
- **non-pointer isa**: 一种 `isa` 指针优化方案，将类对象地址和对象状态信息共同编码进 64 位空间中，用于节省内存并提高运行时效率。
- **内存对齐**: 编译器或运行时按照特定字节边界组织数据的规则，例如对象内部常见 8 字节对齐，系统分配对象空间时常见 16 字节对齐。
- **位域（Bit-field）**: C/C++ 中可以指定字段占用 bit 位数量的语法，例如只用 1 位保存布尔状态，常用于底层源码中节省空间。
- **联合体（Union）**: 多个成员共享同一块内存的 C 语言数据结构。它能节省空间，但同一时间通常只能可靠保存一个成员的有效值。
