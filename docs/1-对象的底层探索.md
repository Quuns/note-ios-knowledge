### 思考 
1. **[为什么 `alloc` 后多次调用 `init` 得到的是同一个对象？](https://bibigpt.co/search?q=%E4%B8%BA%E4%BB%80%E4%B9%88%20%60alloc%60%20%E5%90%8E%E5%A4%9A%E6%AC%A1%E8%B0%83%E7%94%A8%20%60init%60%20%E5%BE%97%E5%88%B0%E7%9A%84%E6%98%AF%E5%90%8C%E4%B8%80%E4%B8%AA%E5%AF%B9%E8%B1%A1%EF%BC%9F)**
  - 因为默认的 `init` 方法并不会重新开辟内存，它只是返回当前对象本身，也就是 `self`。真正负责开辟内存和创建实例对象的是 `alloc` 底层流程中的相关函数，而不是默认 `init`。
- 2. **[为什么源码里能看到某些函数调用，但汇编里却看不到？](https://bibigpt.co/search?q=%E4%B8%BA%E4%BB%80%E4%B9%88%E6%BA%90%E7%A0%81%E9%87%8C%E8%83%BD%E7%9C%8B%E5%88%B0%E6%9F%90%E4%BA%9B%E5%87%BD%E6%95%B0%E8%B0%83%E7%94%A8%EF%BC%8C%E4%BD%86%E6%B1%87%E7%BC%96%E9%87%8C%E5%8D%B4%E7%9C%8B%E4%B8%8D%E5%88%B0%EF%BC%9F)**
  - 这是因为编译器会进行优化。即使在 Debug 模式下，编译器也可能对部分无用变量、简单函数调用或可内联逻辑进行优化，所以源码和汇编并不一定一一对应。
- 3. **[对象为什么实际分配内存时要按 16 字节对齐？](https://bibigpt.co/search?q=%E5%AF%B9%E8%B1%A1%E4%B8%BA%E4%BB%80%E4%B9%88%E5%AE%9E%E9%99%85%E5%88%86%E9%85%8D%E5%86%85%E5%AD%98%E6%97%B6%E8%A6%81%E6%8C%89%2016%20%E5%AD%97%E8%8A%82%E5%AF%B9%E9%BD%90%EF%BC%9F)**
  - 这是为了提升 CPU 读取内存的效率。对象内部至少包含一个 `isa` 指针，占用 8 字节；系统采用 16 字节对齐可以在内存浪费和读取效率之间取得较好的平衡，属于典型的“以空间换时间”。


### 术语解释
- **`alloc`**: Objective-C 中用于创建对象并申请内存的方法。底层会经过消息发送、根类分配逻辑、实例大小计算和内存申请等流程。
- **`init`**: Objective-C 中用于初始化对象的方法。NSObject 默认实现基本只是返回 `self`，通常需要子类重写来完成自定义初始化。
- **`objc_msgSend`**: Objective-C 消息发送机制的核心函数，用于根据接收者和方法名查找并调用对应的方法实现。
- **`isa` 指针**: Objective-C 对象内部的重要指针，用于关联对象与其类信息，是理解对象、类、元类关系的关键。
- **内存对齐**: 为了提高 CPU 访问内存效率，系统按照固定字节边界分配或排列数据。对象常涉及 8 字节和 16 字节对齐，结构体则根据成员类型和最大成员大小进行对齐。
