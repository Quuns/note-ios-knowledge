## 摘要
本节课主要围绕 Objective-C 底层中的“懒加载类、非懒加载类以及分类 Category 的加载流程”展开。课程先回顾了上一节非懒加载类的加载路径，然后通过源码断点、调用栈和编译产物分析，说明懒加载类在首次接收消息时才会触发加载。随后重点讲解 Category 的数据结构、类扩展与分类的区别、关联对象给分类添加属性的原理，以及不同“类与分类是否实现 `+load`”组合下 Category 数据最终进入 `ro`、`rw` 或 `rwe` 的情况。


### 亮点
- 🧭 非懒加载类的加载流程通常从 Objective-C 运行时初始化回调开始，经过 `map_images`、`read_images`，最终进入 `realizeClassWithoutSwift` 等关键函数。
- 🧱 `realizeClassWithoutSwift` 不只负责类本身的初始化，还会递归处理父类和元类，并设置 superclass 与 isa 指针等核心关系。
- 🚀 懒加载类不会在程序启动阶段立即完成完整加载，而是在类第一次接收到消息时，通过消息查找流程触发加载。
- 🔍 当调用类方法或实例方法时，如果缓存中找不到方法，会进入 `objc_msgSend_uncached`，再经过 `lookUpImpOrForward` 等流程触发类的 realize。
- 🧩 Category 编译后会变成 `category_t` 结构体，其中包含分类名、宿主类、实例方法列表、类方法列表、协议列表和属性列表。
- 🧠 Category 本身没有独立的元类，分类中的类方法在编译结构中以列表形式保存，运行时会合并到宿主类相关结构中。
- 🚫 Category 的结构体中没有成员变量存储区域，因此分类原则上不能直接添加成员变量。
- 🔗 分类中声明属性不会自动生成完整可用的 getter 和 setter，需要通过关联对象手动实现属性读写。
- 🧰 `objc_setAssociatedObject` 和 `objc_getAssociatedObject` 的底层依赖全局关联表，通过对象、key、关联策略和值之间的映射完成属性绑定。
- 🧵 设置关联对象时运行时会加锁，目的是保证同一时间只有一个线程操作关联对象的全局哈希表。
- 🗑️ 被关联对象释放时，运行时会自动移除关联关系，因此一般不需要开发者手动管理关联对象生命周期。
- 🆚 类扩展也叫匿名分类，它必须写在类声明和类实现之间，可以添加属性、方法和成员变量，且依赖本类实现。
- 🧾 Category 可以单独实现，主要用于给已有类添加方法；类扩展不能单独实现，通常用于声明私有属性、私有方法或私有成员变量。
- ⚙️ 当类和分类都实现 `+load` 时，它们都属于非懒加载场景，运行时更容易产生 `rwe` 结构，带来额外内存开销。
- 📌 如果类和分类存在同名方法，分类方法通常会被插入到方法列表前面，因此消息查找时会优先命中分类方法。
- 🧮 如果多个分类都实现了同名方法，通常后编译的分类方法会更靠前，因此更容易被优先调用。
- 💡 课程最后提醒，除非确实需要，不建议随意给类和分类都实现 `+load`，因为这可能增加运行时内存和初始化成本。

### 思考 
1. **[Objective-C 中懒加载类到底什么时候会被加载？](https://bibigpt.co/search?q=Objective-C%20%E4%B8%AD%E6%87%92%E5%8A%A0%E8%BD%BD%E7%B1%BB%E5%88%B0%E5%BA%95%E4%BB%80%E4%B9%88%E6%97%B6%E5%80%99%E4%BC%9A%E8%A2%AB%E5%8A%A0%E8%BD%BD%EF%BC%9F)**
  - 懒加载类通常不会在程序启动阶段立即完成加载，而是在第一次接收到消息时触发加载。比如调用类方法、实例方法或创建对象时，消息发送流程会进入方法查找，如果缓存中找不到实现，就可能通过 `lookUpImpOrForward` 等函数触发 `realizeClassWithoutSwift`。
- 2. **[为什么 Category 不能直接添加成员变量？](https://bibigpt.co/search?q=%E4%B8%BA%E4%BB%80%E4%B9%88%20Category%20%E4%B8%8D%E8%83%BD%E7%9B%B4%E6%8E%A5%E6%B7%BB%E5%8A%A0%E6%88%90%E5%91%98%E5%8F%98%E9%87%8F%EF%BC%9F)**
  - 因为 Category 编译后的 `category_t` 结构体中没有用于存储成员变量的字段。它主要保存方法列表、协议列表和属性列表等信息，所以分类声明属性时不会自动拥有成员变量，需要通过关联对象模拟属性存储。
- 3. **[分类和本类有同名方法时，为什么通常会执行分类的方法？](https://bibigpt.co/search?q=%E5%88%86%E7%B1%BB%E5%92%8C%E6%9C%AC%E7%B1%BB%E6%9C%89%E5%90%8C%E5%90%8D%E6%96%B9%E6%B3%95%E6%97%B6%EF%BC%8C%E4%B8%BA%E4%BB%80%E4%B9%88%E9%80%9A%E5%B8%B8%E4%BC%9A%E6%89%A7%E8%A1%8C%E5%88%86%E7%B1%BB%E7%9A%84%E6%96%B9%E6%B3%95%EF%BC%9F)**
  - 运行时合并方法列表时，分类方法通常会被插入到原有方法列表前面。消息查找是按方法列表顺序查找的，一旦找到匹配方法就返回，因此分类中的同名方法会优先被调用。


### 术语解释
- **懒加载类（Lazy Class）**: 指不会在程序启动阶段立即完整加载的类，通常在第一次接收到消息时才触发运行时加载和初始化。
- **非懒加载类（Non-Lazy Class）**: 通常指实现了 `+load` 方法的类或分类，它们会在程序启动阶段被运行时提前处理。
- **Category（分类）**: Objective-C 中给已有类扩展方法的一种机制。分类可以添加实例方法、类方法、协议和属性声明，但不能直接添加成员变量。
- **关联对象（Associated Object）**: Runtime 提供的一套机制，用于给对象动态绑定额外数据，常用于给 Category 实现“属性”功能。
- **`ro` / `rw` / `rwe`**: Objective-C 类底层数据结构中的不同层次。`ro` 通常表示编译期只读信息，`rw` 表示运行期可变信息，`rwe` 常用于承载额外扩展信息，例如分类或运行时动态修改后的数据。
