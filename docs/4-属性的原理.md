## 摘要
- 本节课主要深入探讨了 iOS 底层类与对象的存储原理。课程详细解析了类（Class）中 `class_rw_t` 和 `class_ro_t` 结构体的区别与联系，揭示了成员变量、实例方法及类方法的内存分布。重点阐述了苹果通过“只读内存（Clean Memory）”与“读写内存（Dirty Memory）”的分层设计，以及如何通过延迟加载和动态重排优化内存占用，特别是针对动态修改类信息（如分类、Runtime API 调用）时的内存管理机制。


### 亮点
- 🍎 **成员变量存储**：真正的成员变量存放在类对象中的 `class_ro_t` 结构体里，苹果不会对成员变量进行重排，且成员变量一旦定义即不可动态修改。
- 📦 **内存优化策略**：苹果将类的数据分为 `class_ro_t`（只读，存编译期信息）和 `class_rw_t`（读写，存运行期动态修改信息），通过按需生成 `rw_ext` 结构体，有效减少了内存消耗。
- ⚙️ **类方法位置**：类方法并非存放在类对象中，而是存储在元类（Meta Class）内，这通过统一了实例方法与类方法的调用流程，极大地复用了消息发送机制。
- 🚀 **消息发送效率**：设计元类的根本目的是为了“复用消息机制”，通过 `isa` 指针指向元类查找类方法，避免了在 `objc_msgSend` 中进行复杂的对象类型判断。
- 🛠️ **Runtime 探索**：通过 `class_copyIvarList`、`class_copyPropertyList` 等 API 可以动态获取类的信息，并解析属性编码（Type Encoding），揭示了底层方法实现（IMP）与选择器（SEL）的本质。

### 思考
1. **[为什么苹果要设计元类（Meta Class）？](https://bibigpt.co/search?q=%E4%B8%BA%E4%BB%80%E4%B9%88%E8%8B%B9%E6%9E%9C%E8%A6%81%E8%AE%BE%E8%AE%A1%E5%85%83%E7%B1%BB%EF%BC%88Meta%20Class%EF%BC%89%EF%BC%9F)**
  - 为了复用消息发送机制（`objc_msgSend`）。设计元类可以将类方法统一纳入实例方法的查找逻辑，使得消息发送时无需判断接收者是类对象还是实例对象，从而简化底层逻辑并提高执行效率。
2. **[`class_ro_t` 和 `class_rw_t` 的核心区别是什么？](https://bibigpt.co/search?q=%60class_ro_t%60%20%E5%92%8C%20%60class_rw_t%60%20%E7%9A%84%E6%A0%B8%E5%BF%83%E5%8C%BA%E5%88%AB%E6%98%AF%E4%BB%80%E4%B9%88%EF%BC%9F)**
  - `class_ro_t`（Read-Only）是在编译期生成的，存储了类不可变的结构信息（如名称、方法列表等），属于 Clean Memory；`class_rw_t`（Read-Write）是在运行期生成的，存储了类在动态运行时可能发生变化的数据，属于 Dirty Memory。


### 术语解释
- **Clean Memory**: 只读内存，存储类编译后的原始数据，当内存不足时可被系统移除，之后再从磁盘重新加载。
- **Dirty Memory**: 读写内存，在程序运行期间被修改过，必须常驻内存，因此苹果尽量减少这部分数据的大小。
- **Type Encoding**: 在 Runtime 中用于描述变量、参数或返回值类型的字符串编码（如 `i` 代表 `int`，`@` 代表对象），它是 `method_t` 和 `property_t` 的重要组成部分。
- **元类 (Meta Class)**: 专门用于存放类方法的数据结构，其本质也是一个类对象，拥有自己的 `isa` 指针。
